Day 1 · TryHackMe — Advent of Cyber 2025
Linux CLI — Shell Bells
Linux · CLI
Problem Statement

The TBFC SOC systems had been accessed without authorisation. The task was to investigate the compromise by navigating the Linux file system using only the command line — finding hidden files, reading logs, and uncovering what the attacker left behind.

Approach

Used Linux CLI commands to navigate directories, list hidden files, and read log files. Investigated bash history and environment variables to locate hidden credential fragments (PRESSFRAGs) left by the attacker, then combined them to reconstruct an access passcode.

Tools Used
Linux CLIls / cd / catbash history.bashrc inspectiongrep
Key Lessons Learned
Bash history and environment variables can reveal sensitive information left by attackers or careless users. Hidden files (dotfiles) and shell config files like .bashrc are common places to hide or inadvertently expose credentials. Navigating Linux confidently from the CLI is a fundamental defensive skill.
Day 3 · TryHackMe — Advent of Cyber 2025
Splunk Basics — Did You SIEM?
SIEM · Log Analysis
Problem Statement

The TBFC SOC needed to identify a security incident by searching through large volumes of log data. The challenge required using Splunk to ingest, search, and correlate logs to detect malicious activity within the Wareville network.

Approach

Navigated the Splunk interface to search and filter log data using SPL (Search Processing Language). Built queries to correlate events, filter by time range and source IP, and identify patterns consistent with an intrusion. Used field extraction and stats commands to surface anomalous behaviour.

Tools Used
SplunkSPL (Search Processing Language)Log AnalysisEvent Correlation
Key Lessons Learned
SIEM platforms like Splunk are essential for defenders — the ability to write effective SPL queries directly impacts how quickly a SOC can detect and respond to incidents. Log data is only useful when you know how to query it efficiently. Proper indexing and field extraction make the difference between finding an attacker in minutes vs hours.
Day 7 · TryHackMe — Advent of Cyber 2025
Network Discovery — Scan-ta Clause
Networking · Recon
Problem Statement

HopSec had breached TBFC's QA environment and locked everyone out. The target server (IP: 10.67.173.8) was compromised — its website defaced and services locked. The objective was to perform network discovery to identify all open ports and services, exploit misconfigurations to regain access, and recover hidden keys from multiple services.

Approach

Ran a full Nmap port scan against the target to enumerate open TCP and UDP ports. Identified services including SSH (22), HTTP (80), DNS (53), FTP (21), and a custom app on port 25251. Discovered anonymous FTP access and an unauthenticated TBFC maintenance service. Exploited a locally accessible MySQL database (3306) — which had no password — to retrieve the final flag from the flags table.

Tools Used
NmapFTP clientMySQL CLIWeb browserAttackBox (Ubuntu VM)
Key Lessons Learned
Anonymous FTP access and passwordless local database connections are critical misconfigurations that attackers exploit routinely. A thorough port scan is always the first step in any engagement — services running on non-standard ports are easily missed without a full scan. Defence-in-depth means never assuming local access is safe: databases must require authentication regardless of their bind address.
